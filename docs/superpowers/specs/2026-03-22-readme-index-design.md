# README 资料索引设计

## 背景

当前仓库根目录下的 `README.md` 只有标题, 还没有清晰展示仓库中已有资料的分类与入口。用户希望将 README 完善为可直接浏览的索引页, 并明确要求按类别展示表格, 其中文章类还需要给出原文链接。

## 目标

- 将 `README.md` 改成面向浏览的资料导航页
- 使用 4 个二级标题分别承载 4 类内容
- 每个二级标题下包含 1 句简要说明和 1 个表格
- 文章类表格提供原文链接
- 保持内容简洁, 方便后续持续追加资料

## 分类方案

采用以下 4 类结构:

1. 文章
2. Prompt
3. Skill
4. 规范

分类依据如下:

- `artical/` 下文件属于阅读资料与参考文章, 归入“文章”
- `prompt/` 下文件属于可复用提示词模板, 归入“Prompt”
- `skill/` 下文件属于技能说明、模板或脚手架, 归入“Skill”
- `AGENT.md` 属于项目协作与使用约定, 归入“规范”
- 最终整理时以仓库当前实际文件路径为准, 分类名称需要与实际目录或文件一一对应

## 收录范围

本次 README 整理仅覆盖当前仓库内已经存在、且适合作为资料入口展示的内容:

- 文章: `artical/` 目录下现有 7 篇文章文件
- Prompt: `prompt/` 目录下现有 2 份提示词文件
- Skill: `skill/` 目录下现有技能说明与模板文件
- 规范: 当前先收录根目录下的 `AGENT.md`

本次不将 `.git/` 内部文件和其他非资料型文件纳入 README 索引。

## README 结构设计

README 顶部保留一个简短总说明, 用来说明仓库用途。

README 正文中的 4 个分类一律使用二级标题, 不使用同级的其他分类标题。

各分类的展示方式如下:

### 文章

- 简要说明: 说明这里收录 Agent 相关学习资料与参考文章
- 表格列:
  - 名称
  - 简介
  - 原文链接
  - 本地文件

### Prompt

- 简要说明: 说明这里收录可复用提示词模板
- 表格列:
  - 名称
  - 用途
  - 本地文件

### Skill

- 简要说明: 说明这里收录技能说明与模板文件
- 表格列:
  - 名称
  - 用途
  - 本地文件

### 规范

- 简要说明: 说明这里收录协作约定与项目规则
- 表格列:
  - 名称
  - 用途
  - 本地文件

“规范”这一类用于收录项目协作约定、使用规则和流程限制文件。当前先收录 `AGENT.md`, 后续若新增同类规则文件, 也继续归入这一类。

## 内容整理原则

- 仅整理当前仓库中已存在的资料, 不新增虚构条目
- 文章简介以简短概括为主, 不展开长篇描述
- 原文链接优先从文章正文中已有链接提取
- 文章类以当前 `artical/` 目录内文件为收录范围, 实现时需为每篇文章补齐原文链接
- 若个别文章的原文链接无法从现有正文直接确认, 则在实现阶段继续查找其对应来源, 直到可以补入 README 后再完成本次整理
- 对于没有原文链接的非文章类资料, 不额外补造外部来源
- 本地文件统一使用相对路径, 方便点击跳转

## 验收标准

- `README.md` 中存在 4 个二级标题
- 每个二级标题下都有 1 句说明和 1 个表格
- 文章类表格中的每条文章都有原文链接列
- README 可以作为仓库资料总入口直接使用
